# Packages View (replaces Unit Templates)

## Summary
Renamed "Unit Templates" → "Packages" in the explorer dropdown and replaced the
placeholder tree-view (which scanned a non-existent `unit_templates/` directory) with
a rich card-based view that reflects the actual package structure in `_modules/`.

---

## Changes — homelab_gui/homelab_gui.py

### `_scan_packages()` (new, replaces unit_templates scan in `_init_modules_cache`)
Builds a list of package dicts from three sources:
- `_modules/<provider>/<package>/<module>/` — discovers packages, their providers, and module .tf paths
- Stack config YAMLs (`_find_stack_configs()`) — attaches config_yaml_path per package
  (`terragrunt_lab_stack.yaml` → `default`, `terragrunt_lab_stack_<pkg>.yaml` → `<pkg>`)
- `scripts/tg-scripts/<pkg>/` and `scripts/wave-scripts/<pkg>/` — attaches links to
  `pkg_env.sh` and `README.md` in each scripts directory

Each package dict contains:
`name`, `config_yaml_path`, `providers_str`, `modules` (list of `{provider, name, tf_path}`),
`tg_scripts_env_path`, `tg_scripts_readme_path`, `wave_scripts_env_path`, `wave_scripts_readme_path`.
All path fields are absolute path strings; empty string means "not found".

### `_PACKAGES_CACHE` (replaces `_UNIT_TEMPLATES_NODES_CACHE`)
Module-level cache populated by `_init_modules_cache()` via `_scan_packages()`.

### State vars
- Removed: `unit_templates_nodes`, `unit_templates_expanded_paths`
- Added: `packages_data: list[dict]`

### Computed vars
- Removed: `unit_templates_nodes_empty`, `visible_unit_templates_nodes`
- Added: `packages_data_empty`

### `open_abs_file_in_viewer(abs_path)` (new event handler, replaces `click_unit_templates_node`)
Opens any absolute file path in the file viewer:
- Sets `file_viewer_mode` to `"config_data"` for `.yaml`/`.yml`, else `"unit_file"`
- Sets `hcl_content` and `hcl_file_path`
- Returns `_search_reapply_script()` to keep search marks live

### UI components
- Removed: `unit_templates_node_component`, `unit_templates_tree_view`
- Added:
  - `_pkg_module_row(mod)` — single module row (provider | clickable module name)
  - `_pkg_script_link(label, path)` — clickable script-file pill (hidden when path is empty)
  - `_pkg_card(pkg)` — full card for one package: header (name + config YAML link),
    providers row, modules list with .tf links, TG/wave scripts links
  - `packages_view()` — scrollable view with a `rx.foreach` over `AppState.packages_data`

### Strings / values
- `"unit_templates"` explorer_root value → `"packages"`
- `"Unit Templates"` menu text → `"Packages"`; label → `"Packages ▾"`
- Status print: `"{n} unit templates"` → `"{n} packages"`

---

## Result
The Packages dropdown entry now shows one card per package (currently: `default`, `demo-cloud-buckets`).
Each card includes clickable links to the config YAML, each module's main.tf, and
pkg_env.sh / README.md in both the TG-scripts and wave-scripts directories — all of which
open in the file viewer on click.
