# 20260403-021214 — Add "Open Kube Shell" context menu action for kubeconfig nodes

## What changed

`homelab_gui/homelab_gui.py` — `open_context_menu` method (line ~6371):

Added detection of `kubeconfig` nodes (last path segment == `"kubeconfig"`) and injection
of an **"Open Kube Shell"** action into the Shell group of the context menu.

When the action is triggered, it reuses the existing `ssh` action_type (→ `open_ssh_terminal`)
with `initial_cmd`:
```
export KUBECONFIG=$_DYNAMIC_DIR/kubeconfig/{cluster_name}_kubeconfig.yaml && kubectl get nodes
```
where `cluster_name` is the second-to-last path segment (e.g. `gke-cluster-dev` from
`cat-hmc/gcp/us-central1/dev/clusters/gke-cluster-dev/kubeconfig`).

The terminal inherits the user's shell environment, so `$_DYNAMIC_DIR` resolves at
runtime in the spawned bash process.

## No new state vars, no dispatch_action changes needed

The `kube_shell` action reuses `action_type: "ssh"` which is already handled by
`dispatch_action` → `open_ssh_terminal(cwd, cmd)`. No new elif branch needed.
