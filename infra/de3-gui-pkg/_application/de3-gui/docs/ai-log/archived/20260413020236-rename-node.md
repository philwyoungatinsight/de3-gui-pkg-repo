# 20260413020236 — Add rename-node to infra tree view

## What changed

Added in-place rename capability to the infra folder tree context menu.
Single file changed: `homelab_gui/homelab_gui.py`.

### New state variables (AppState)
- `rename_dialog_open`, `rename_pending_path`, `rename_pending_name`, `rename_error`

### New module-level helpers
- `_rename_in_config_params_dict(cp, old_path, new_path) → bool`  
  Renames all keys in a `config_params` dict that exactly match or start with `old_path`.
- `_rename_config_keys_in_yaml_files(old_path, new_path) → int`  
  Scans every `infra/*/_config/*.yaml` (excluding SOPS files), rewrites matching
  `config_params` keys in both the flat (`pkg.config_params`) and legacy
  (`pkg.providers.<prov>.config_params`) formats.
- `_rename_config_keys_in_sops_file(sops_file, old_path, new_path) → bool`  
  Decrypts the SOPS file via `sops --decrypt`, rewrites matching keys,
  writes a temp plaintext file in the **same directory** as the SOPS file
  (so `.sops.yaml` creation rules apply), then atomically re-encrypts
  with `sops --encrypt --output`.

### New AppState event handlers
- `begin_rename(path)` — opens dialog pre-filled with the current last-segment name
- `set_rename_name(name)` — updates input + clears error
- `rename_name_keydown(key)` — Enter submits
- `cancel_rename()` — closes dialog
- `confirm_rename()` — validates, rewrites YAML + SOPS, `mv`s the directory,
  invalidates the SOPS cache, reloads the node tree and updates `selected_node_path`

### Context menu
- New `"edit"` group added between `"destroy"` and `"clipboard"` with a single
  `"Rename…"` item available on every node.
- `group_labels` extended with `"edit": "Edit"`.
- `dispatch_action` extended with `elif action_type == "begin_rename"`.

### Dialog UI
- Rename dialog added after the delete-confirmation dialog:
  shows the current path, a description of what will be updated, a monospace
  text input (auto-focused, Enter to submit), an inline validation error line,
  Cancel + Rename buttons.

## Design decisions
- Validation rejects empty names, names containing `/`, and `.`/`..`.
- Validation errors keep the dialog open (set `rename_error`); the dialog only
  closes when the operation succeeds or the user cancels.
- SOPS re-encryption uses a temp file in `sops_file.parent` so the repo-root
  `.sops.yaml` path_regex matches and the correct PGP keys are selected.
- `_SOPS_SECRETS_CACHE` / `_SOPS_SECRETS_LOADED` are reset after a successful
  rename so secrets reload with the updated keys on next access.
- The `selected_node_path` is updated to the new path if the renamed node (or an
  ancestor of the selected node) was the one being renamed.
- SOPS failures are non-fatal: reported as warnings in `clipboard_message` but
  the rename still proceeds (the directory and YAML files will have been updated).
