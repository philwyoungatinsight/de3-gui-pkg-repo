# 20260403-021525 — Fix Kube Shell: source set_env.sh + add button to browser panel

## What changed

### 1. `_build_kube_shell_cmd(cluster_name)` helper added (near `_get_ssh_command`)
Extracts the kube shell command into a reusable function.  
Command now sources `set_env.sh` first so `$_DYNAMIC_DIR` is available:
```bash
source $(git rev-parse --show-toplevel)/set_env.sh \
  && export KUBECONFIG=$_DYNAMIC_DIR/kubeconfig/{cluster_name}_kubeconfig.yaml \
  && kubectl get nodes
```

### 2. `open_context_menu` — uses `_build_kube_shell_cmd` (no behaviour change)

### 3. `selected_node_browser_actions` — "Kube" button injected
When `selected_node_path` ends with `"kubeconfig"`, a **"Kube"** button is now
injected into the browser-panel header action buttons (same row as "Shell" / "SSH").
Uses `action_type: "ssh"` → `open_ssh_terminal` with the same sourced command.
