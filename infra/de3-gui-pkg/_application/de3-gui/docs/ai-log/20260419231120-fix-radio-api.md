# Fix rx.radio / rx.radio_group API mismatch (Reflex 0.8.27)

`homelab_gui.py:16692` called `rx.radio("skip")` and `rx.radio_group(...)` using the old-style Radix
radio API. In Reflex 0.8.27, `rx.radio` and `rx.radio_group` both resolve to `HighLevelRadioGroup`,
which expects a list of values rather than acting as a single item or root container.

Fix: replaced `rx.radio_group(...)` with `rx.radio.root(...)` and each `rx.radio("value")` with
`rx.radio.item(value="value")` — the low-level Radix namespace that was always present.

App now starts and serves at http://localhost:9080/.
